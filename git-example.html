1. Core Concepts

Understand the difference between Git (local version control) and GitHub (cloud collaboration platform), along with the foundational repository lifecycle stages.

Git vs GitHub

Git is local tracking software installed on your machine. GitHub is a cloud platform for remote hosting, collaboration, and pull-request workflows.

The Three States

  • Working Directory: Untracked or modified files currently being edited.
  • Staging Area: Preparing and marking changes to be included in next commit.
  • Local Repository: Sealed snapshots safely stored history.

4. Best Practices

Use Semantic Messages

Prefix commit messages with tags: feat:, fix:, docs:, refactor:. E.g., feat: add login toggle.

Protect Secrets

Always add credentials, dependencies, and environment files to your local .gitignore file before staging files.

Stay Rebased & Updated

Execute git pull regularly to synchronize local repositories and minimize complicated merge conflicts.

Keep Commits Focused

A single commit should only tackle one task or feature. Atomic commits make rollbacks and version history browsing straightforward.
